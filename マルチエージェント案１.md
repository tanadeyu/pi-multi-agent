# マルチエージェント構成 案1（最終版）

> pi + tmux + pi-gl-mem によるミニマルマルチエージェント（編集者・ライター・レビュアー）

## ディレクトリ構造

```
project/                          ← ユーザー専用（直接触る場所）
│
├── editor/                       ← 編集者エージェント
│   ├── .pi-gl-mem/               # 記憶（役割・設計・品質基準）・自動ロード
│   │   └── MEMORY.md             # 「あなたは編集者です…」と役割を書く
│   ├── inbox/                    ← ユーザー・ライター・レビュアーからの入力
│   └── outbox/                   ← 自分が送ったものの記録
│
├── writer/                       ← ライターエージェント
│   ├── .pi-gl-mem/MEMORY.md      # 「あなたはライターです…」
│   ├── inbox/
│   └── outbox/
│
├── reviewer/                     ← レビュアーエージェント
│   ├── .pi-gl-mem/MEMORY.md      # 「あなたはレビュアーです…」
│   ├── inbox/
│   └── outbox/
│
├── mini-agent.sh                 # tmux 4ペイン起動 + pipe-pane ログ
└── logs/                         # 各ペインの出力ログ（pipe-pane）
```

- 全エージェント `.pi-gl-mem/pi_gl_settings.json` → `{"injectGlobal": false}`
- 各 `.pi-gl-mem/MEMORY.md` に役割を書けば自動ロードされる（別途 SYSTEM.md 不要）

## 連携ルール（これだけ）

### タスクの流れ

```
① inbox を確認 → 未処理ファイルがあれば着手
② 作業
③ 新規ファイルを同時に2つ作る
   ├── 自分の outbox/ に送信記録
   └── 相手の inbox/ に依頼/報告
④ 自分の inbox の処理済みファイルは削除
⑤ ①に戻る
```

### ファイル命名規則

```
YYYY-MM-DD-HHmm_from-送信者_to-受信者_件名.md
例: 2026-06-25-1402_from-editor_to-writer_ログイン機能.md
```

- 毎回 **新規ファイル**（既存ファイルの修正はしない）
- 同時書き込みが原理的に発生しない

### 誰がどこにファイルを置くか

| 送信者 | 自分の outbox/ に作成 | 相手の inbox/ に作成 |
|--------|----------------------|---------------------|
| ユーザー | （任意） | `editor/inbox/` |
| 編集者 | `editor/outbox/` | `writer/inbox/` または `reviewer/inbox/` |
| ライター | `writer/outbox/` | `reviewer/inbox/`（成果物）または `editor/inbox/`（報告） |
| レビュアー | `reviewer/outbox/` | `writer/inbox/`（フィードバック）または `editor/inbox/`（承認報告） |

### ライター↔レビュアーのループと完了

```
ライター → reviewer/inbox/ に成果物
レビュアー → writer/inbox/ に修正指摘（ループ継続）
レビュアー → editor/inbox/ に「承認しました」（← これが完了の表現）
```

- レビュアーが `editor/inbox/` に承認報告を送った＝両者納得

## 起動スクリプト（mini-agent.sh）

```bash
#!/bin/bash
SESSION="writing-team"
PROJECT="/path/to/project"
LOG="$PROJECT/logs"

mkdir -p "$LOG"

tmux new-session -d -s "$SESSION" -c "$PROJECT/editor" -n editor
tmux pipe-pane -t "$SESSION:editor" -o "cat >> $LOG/editor.log"
tmux send-keys -t "$SESSION:editor" "cd $PROJECT/editor && pi" Enter

tmux new-window -t "$SESSION" -n writer -c "$PROJECT/writer"
tmux pipe-pane -t "$SESSION:writer" -o "cat >> $LOG/writer.log"
tmux send-keys -t "$SESSION:writer" "cd $PROJECT/writer && pi" Enter

tmux new-window -t "$SESSION" -n reviewer -c "$PROJECT/reviewer"
tmux pipe-pane -t "$SESSION:reviewer" -o "cat >> $LOG/reviewer.log"
tmux send-keys -t "$SESSION:reviewer" "cd $PROJECT/reviewer && pi" Enter

tmux new-window -t "$SESSION" -n user -c "$PROJECT"
tmux pipe-pane -t "$SESSION:user" -o "cat >> $LOG/user.log"

tmux attach -t "$SESSION"
```

## エージェントに教えるルール（AGENTS.md）

```markdown
## 共通ルール
- 起動時に自分の inbox/ のファイルをすべて確認すること
- 未処理ファイルがあれば内容に従い直ちに着手すること
- 作業がひと段落したら、自分の outbox/ と相手の inbox/ に新規ファイルを作成すること
- ファイル名は YYYY-MM-DD-HHmm_from-差出人_to-宛先_件名.md とすること
- 処理済みの inbox ファイルは削除して良い
- タスクが無ければ待機（無駄なAPIコストをかけない）
```
