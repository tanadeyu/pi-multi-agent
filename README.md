# pi-multi-agent

> **pi + tmux + pi-gl-mem** によるパイプライン型マルチエージェントシステム

---

## 現在のバージョン

**v2.3.0** — 固定3エージェント ミニマム検証版

過去バージョンは [`oldfiles/`](./oldfiles/) に保管されています。

---

## 概要

pi（非対話モード `pi -p`）+ tmux + pi-gl-mem によるマルチエージェントシステム。
固定3エージェント（manager → writer → reviewer → manager）がファイルベースで連鎖し、
最終成果物に至るまで自動実行する。

**目的**: シングルエージェント vs マルチエージェントのハルシネーション比較検証

---

## クイックスタート（推奨）

固定3エージェント版（最小構成）:

```bash
cd /mnt/c/projects/pi-multi-agent
bash pi-team-minimal.sh <チーム名> "タスク内容"
```

例:

```bash
bash pi-team-minimal.sh test "2025年の日本の半導体補助金の総額と主な使途を教えてください"
```

これで tmux 3ペインが起動し、manager → writer → reviewer の連鎖が自動実行されます。

### 従来版（パイプライン方式 v2.2.0）

```bash
bash pi-team-run.sh
  → チーム名を入力
  → ワーカー数 (1-4) を入力
  → tmux セッションが起動
```

## アーキテクチャ

### 固定3エージェント（pi-team-minimal.sh）

```
manager（統括・最終判断）
  ↓ 指示ファイル
writer（ドラフト生成）
  ↓ 成果ファイル
reviewer（ファクトチェック・スコアリング）
  ↓ 承認 or 差し戻し
manager（完了）
```

- 全エージェント `pi -p`（非対話）で起動
- 連鎖は `tmux send-keys` で自動実行
- timeout 180秒（ハング防止）
- リビジョン管理（v1/v2/v3）で無限ループ防止

### reviewer スコアリング基準

| 評価軸 | 配点 | 判定基準 |
|--------|------|---------|
| ソース信頼性 | 0-40 | 一次ソース=40、二次ソース=20、なし=0 |
| 最新性 | 0-30 | 現在日付比較、年月未記載で減点 |
| 論理・一貫性 | 0-20 | 内部矛盾なし=20 |
| 表現の慎重さ | 0-10 | 過度な断定を減点 |

合格ライン: **85点以上** 承認 / 70-84点 注記 / **70点未満** 差し戻し
