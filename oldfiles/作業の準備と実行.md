# 作業の準備と実行

## 前提
- tmux がインストールされていること
- `timeout` コマンドが使えること（GNU coreutils）
- `jq` がインストールされていること（設定ファイルの読み込みに使用）
  ```bash
  sudo apt install -y jq
  ```

## 使い方

### 起動
```bash
bash pi-team-run.sh team_a
```
Windows Terminal が別窓で開き tmux が起動する（今のターミナルは占有されない）。
設定 `team_a.json` が存在しなければ自動生成してから起動する。

### 内部で行われること（生成時）
1. `team_a.json` を作成（セッション名・タイムアウト・エージェント定義・ループ設定）
2. `teams/team_a/` に各エージェントのディレクトリ・`.pi-gl-mem`・`MEMORY.md` を生成
3. 最初のエージェントの `inbox/` に初期タスクを配置
4. Windows Terminal 別窓で tmux 起動（縦 N 段レイアウト）

## tmux 起動後の流れ

### 1. 初期タスクを投入（生成時に自動投入済み）
```bash
# 手動でタスクを作る場合
cat > editor/inbox/2026-06-25-XXXX_from-user_to-editor_タスク名.md << 'EOF'
# タスク内容
EOF
```

### 2. editor にトリガーを送信
```bash
# この pi セッションから（tmux の外）
tmux send-keys -t writing-team:agents.0 \
  "cd teams/team_a/editor && pi -p 'inbox を確認して最初のタスクを処理してください'" Enter
```

### 3. あとは自動連鎖
```
editor → writer → reviewer → editor（承認報告）
```
各エージェントは完了後、自動で `tmux send-keys` で次を起動する。

### 4. 経過確認
```bash
tail -f logs/editor.log
tail -f logs/writer.log
tail -f logs/reviewer.log
```

### 5. tmux の終了
```bash
tmux kill-session -t writing-team
```

## 設定ファイル（team.json）

生成される設定例:
```json
{
  "session": "writing-team",
  "timeout": 180,
  "workflow": "editor(0)→writer(1)→reviewer(2)→editor(0)",
  "loops": [
    { "between": ["reviewer", "writer"], "min": 1, "max": 3 }
  ],
  "agents": [
    { "name": "editor", "role": "あなたは編集者..." },
    { "name": "writer", "role": "あなたはライター..." },
    { "name": "reviewer", "role": "あなたはレビュアー..." }
  ]
}
```

編集可能な項目:
| 項目 | 意味 | 初期値 |
|------|------|--------|
| `session` | tmux セッション名 | `writing-team` |
| `timeout` | 各エージェントのタイムアウト秒 | `180` |
| `workflow` | ワークフロー定義（自由記述） | 循環チェーン |
| `loops[].between` | ループするエージェントペア | `["reviewer","writer"]` |
| `loops[].min` | 最低ループ回数 | `1` |
| `loops[].max` | 最大ループ回数 | `3` |
| `agents` | エージェント定義（名前・役割） | editor / writer / reviewer |

## 補足

### ペイン番号
定義順に割り当てられる（標準: editor=0 / writer=1 / reviewer=2）

### ファイル命名規則
```
YYYY-MM-DD-HHmm_from-送信者_to-受信者_件名.md
```

### クリーンアップ（再実行時）
```bash
rm -f editor/inbox/*.md editor/outbox/*.md
rm -f writer/inbox/*.md writer/outbox/*.md
rm -f reviewer/inbox/*.md reviewer/outbox/*.md
```
